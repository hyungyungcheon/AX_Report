# AX 인텔리전스 리포트 생성기

키워드나 지시문을 입력하면 **최근 기사를 웹에서 수집·요약**하고, **엔터프라이즈 AX(AI Transformation) 관점의 인사이트 리포트**를 생성해 **게시판 형태의 웹 페이지**에 쌓아 주는 프로그램입니다.

---

## 🚀 빠른 시작 (이것만 따라 하세요)

### 1단계 — 프로그램 실행

폴더 안의 **`AX리포트_시작.bat`** 파일을 **더블클릭**합니다.

- 검은 명령창이 하나 뜨고, 잠시 후 브라우저에 **게시판 페이지**가 자동으로 열립니다.
- 검은 창은 서버입니다. **사용하는 동안 닫지 마세요.** (다 쓰고 나서 닫으면 됩니다)

### 2단계 — 키워드 입력

게시판 상단의 **[새 리포트 생성]** 입력창에 궁금한 주제를 입력합니다.

- 단순 키워드: `생성형 AI 도입`, `Claude`, `AI 반도체`
- 문장/지시문도 가능: `국내 제조업의 AI 전환 사례 위주로 조사해줘`
- 옆의 선택 상자에서 **수집 기간**(최근 7/30/90일)과 **기사 수**(8/10/12건)를 고를 수 있습니다.

### 3단계 — [리포트 생성] 버튼 클릭

- 버튼을 누르면 AI가 웹 검색으로 기사를 수집·분석합니다. **보통 3~7분** 걸립니다.
- 진행 상황이 입력창 아래에 실시간으로 표시됩니다. 기다리는 동안 다른 리포트를 읽어도 됩니다.
- 완료되면 **자동으로 새 리포트 페이지가 열립니다.**

### 4단계 — 리포트 보기

- 리포트 페이지에는 **스코어카드**(전환 속도/산업 파급력/도입 성숙도), **핵심 메시지**, 그리고 두 개의 탭이 있습니다:
  - **기사 요약** 탭 — 수집된 기사들의 요약과 원문 링크
  - **AX 인사이트** 탭 — 기회 / 리스크 / 전략 / 트렌드 4개 관점의 분석
- **← 목록으로**를 누르면 게시판으로 돌아갑니다. 만든 리포트는 계속 쌓입니다.

### 종료

검은 명령창을 닫으면 서버가 종료됩니다. 이미 만든 리포트는 모두 저장되어 있으므로, 다음에 `AX리포트_시작.bat`을 다시 실행하면 그대로 다시 볼 수 있습니다.

## 필요 환경

| 항목 | 설명 |
|---|---|
| Python 3.10+ | `python --version` 으로 확인 |
| Claude Code CLI | `claude --version` 으로 확인. 기사 검색·요약·분석에 사용되며, **별도 API 키는 필요 없습니다** (Claude Code 로그인 세션을 그대로 사용). |

> 리포트 1건 생성 시 Claude 사용량이 소모됩니다 (웹 검색 + 분석 1회).

## 터미널(명령줄)에서 쓰는 방법

웹 UI 대신 명령줄로도 모든 기능을 쓸 수 있습니다. PowerShell을 열고 이 폴더에서:

```powershell
# 웹 UI 서버 실행 (= AX리포트_시작.bat 과 동일)
python ax_report.py --serve

# 서버 없이 리포트 1건 바로 생성 (완료 후 브라우저 자동 열림)
python ax_report.py "생성형 AI 도입"

# 옵션 지정
python ax_report.py "Claude" --days 14 --articles 12   # 최근 14일, 기사 12건
python ax_report.py "AI 규제" --model sonnet            # 분석 모델 지정
python ax_report.py "키워드" --no-open                  # 브라우저 열지 않음

# 저장된 리포트 목록 보기
python ax_report.py --list

# 저장된 데이터로 HTML만 다시 생성 (템플릿 수정 후 사용)
python ax_report.py --rebuild
```

## 동작 방식

1. **수집·분석** — Claude CLI 헤드리스 모드(`claude -p`)가 웹 검색(한국어/영어)으로 최근 기사를 수집하고, 기사별 2~3문장 요약과 AX 관점 분석(스코어카드, 핵심 메시지, 기회/리스크/전략/트렌드 인사이트)을 JSON으로 생성합니다.
2. **저장** — 리포트가 `data/posts/<ID>.json` 에 게시글로 저장됩니다 (리포트 1건 = 파일 1개).
3. **렌더링** — `site/index.html`(게시판)과 `site/posts/<ID>.html`(리포트 페이지)이 재생성됩니다.

## 폴더 구조

```
AX_Report/
├── AX리포트_시작.bat     # ★ 더블클릭으로 실행
├── ax_report.py          # 메인 프로그램
├── templates/
│   ├── index.html        # 게시판 템플릿 (입력창 포함)
│   └── report.html       # 리포트 페이지 템플릿
├── data/posts/           # 게시글 원본(JSON) — 백업하려면 이 폴더만 복사
└── site/                 # 생성된 웹 페이지
    ├── index.html        # 게시판
    └── posts/            # 개별 리포트 페이지
```

## 자주 묻는 질문

**Q. 버튼을 눌러도 반응이 없어요.**
서버(검은 창)가 실행 중인지 확인하세요. `site/index.html` 파일을 브라우저로 직접 열면(주소가 `file:///`로 시작) 조회만 가능하고 생성 버튼은 동작하지 않습니다. 반드시 `AX리포트_시작.bat`으로 실행한 뒤 `http://localhost:8940` 주소로 사용하세요.

**Q. 오류가 났어요.**
입력창 아래에 오류 메시지가 표시됩니다. 대부분 잠시 후 다시 시도하면 됩니다. `claude CLI를 찾을 수 없습니다`라고 나오면 Claude Code 설치/로그인을 확인하세요.

**Q. 리포트를 지우고 싶어요.**
게시판에서 각 리포트 행 오른쪽 끝의 **✕ 버튼**을 누르면 됩니다 (확인 창이 뜹니다). 서버 모드에서만 동작하며, 수동으로 지우려면 `data/posts/`에서 해당 JSON 파일을 삭제한 뒤 `python ax_report.py --rebuild`를 실행하세요.

**Q. 디자인을 바꾸고 싶어요.**
`templates/` 안의 HTML을 수정한 뒤 `python ax_report.py --rebuild`를 실행하면 기존 리포트 전체에 반영됩니다.

**Q. 분석 관점을 바꾸고 싶어요.**
`ax_report.py`의 `PROMPT_TEMPLATE`을 수정하세요 (예: 스코어 항목 변경, 인사이트 카테고리 추가).

**Q. 포트 8940이 이미 사용 중이래요.**
`python ax_report.py --serve --port 9000` 처럼 다른 포트를 지정하세요.
